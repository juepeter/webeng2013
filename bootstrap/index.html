Probleme der ersten Kurseinheit
ProblemLösung
In Windows 7 Oracle VM Virtual Box Manager nicht als Administrator installiert, dann treten bei Start der VM Probleme auf Deinstallieren und neu als Admin installieren
USB 2.0-Problem: USB 2.0 is currently enabled for this virtual machine. However, this requires the Oracle VM VirtualBox Extension Pack to be installed. Einfache Lösung: USB 2.0 deaktivieren (Häkchen bei Eigenschaften der VM, USB für USB 2.0 wegnehmen) Grundsätzliche Lösung: Oracle VM VirtualBox Extension Pack installieren. Bei der Erstellung der VM muss das Extension Pack installiert gewesen sein, sonst lässt sich das Häkchen nicht setzen.
PAE-Problem: The guest is trying to switch to the PAE mode which is currently disabled by default in VirtualBox. Eigenschaften der VM: System -> Processor Häkchen bei Enable PAE setzen.
Schlechte Auflösung der Videos Einstellung auf 720p
Webverzeichnis des Webservers, das im Video angegeben wird, existiert nicht Das angegebene Webverzeichnis ist falsch, korrekt ist das Verzeichnis /home/developer/public_html
@ auf Tastatur funktioniert nicht. Dies scheint ein tastaturspezifischer Fehler zu sein. Standardmässig ist Apple-Tastatur in der VM eingestellt Eine simple Lösung ist das Durchprobieren aller Tasten mit AltGr bis man @ findet. Scheint bei fast allen die Kombination AltGrL zu sein. Alternativ kann man auch die Tastatureinstellung ändern (Achtung: wird erst wirksam nach Neustart der VM oder vielleicht reicht auch Neustart der Terminalsession). Der Hersteller meiner Tastatur ist Trust und ich habe willkürlich eine Trust-Tastatur unter Angeboten ausgewählt und es hat funktioniert. Bei anderen hat Tastatur Microsoft Office oder schlicht Tastatur Deutsch funktioniert. Anscheineńd muss man einfach probieren.
Wie funktioniert Kopieren und Einfügen? Generell und im Terminal Generell sollte man in den Einstellungen der VM unter General -> Advanced Shared Clipboard auf Bidirectional setzen. Dann kann man beliebig aus/in X-Fenster der VM kopieren/einfügen mit gewohnten Kontextmenus oder StrgC, StrgV. Aus mir unerfindlichen Gründen geht das im Terminal-Fenster nicht. Hier muss man StrgEinfg für Kopieren und ShiftEinfg für Einfügen verwenden.
Man erhält bei Aufruf der App in der Cloud: 403 Forbidden You don't have permission to access / on this server. Eine genauere Angabe liefert cctrlapp .. log error: Directory index forbidden by Options directive: /app/code/. Ich hatte die index.html aus public_html nach bootstrap verschoben, weil aus dem Text darin eigentlich hervorging, das wir sie als Grundlage benutzen sollen und bootstrap mein Ellipse-Projektverzeichnis ist. Anscheinend wird aber die index.html in public_html benötigt. Merkwürdigerweise war dies bei dem Probelauf mit php-hello_world_app nicht so. Ich arbeite also zur Zeit in meinem Projekt bootstrap mit Ellipse, wenn ich die index.html fertig habe, dann wird sie nach public_html kopiert und dann klappt auch die Veröffentlichung in der Cloud. Für eine Erklärung wäre ich sehr dankbar! Man kann übrigens den 403 Fehler auch lokal produzieren, wenn man die Zugriffsrechte auf index.html zu eng setzt.
